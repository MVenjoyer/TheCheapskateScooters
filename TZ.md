# Лабораторная работа 1
## Техническое задание на разработку системы "TheCheapskateScooters"

### 1. Введение

**1.1. Назначение системы**  
Веб-приложение для аренды тематических самокатов для скуфоф. Система позволяет арендовать самокаты с уникальным дизайном (WOT, пивные банки в колесах и другие креативные варианты).

**1.2. Цели создания**
- Автоматизация процесса аренды тематических самокатов
- Предоставление уникального пользовательского опыта
- Упрощение поиска и бронирования креативных самокатов

### 2. Функциональные требования

**2.1. Просмотр каталога самокатов**
- Отображение списка всех самокатов с их дизайном
- Фильтрация по статусу (доступен/занят)
- Отображение информации о самокате (ID, модель, тип дизайна, статус)

**2.2. Управление арендой**
- **Начало аренды**: Пользователь может взять доступный самокат в аренду
- **Завершение аренды**: Пользователь может завершить активную аренду
- Автоматическое обновление статуса самоката

**2.3. История аренд**
- Просмотр истории всех аренд пользователя
- Отображение деталей аренды (самокат, время начала, время окончания, статус)

### 3. Пользовательские роли

**3.1. Клиент**
- Просмотр каталога самокатов
- Начало и завершение аренды
- Просмотр истории своих аренд

### 4. Бизнес-процессы

**4.1. Процесс "Аренда самоката"**
1. Пользователь просматривает каталог самокатов
2. Выбирает доступный самокат
3. Нажимает кнопку "Арендовать"
4. Система создает запись об аренде
5. Система изменяет статус самоката на "занят"

**4.2. Процесс "Завершение аренды"**
1. Пользователь переходит в раздел "Мои аренды"
2. Находит активную аренду
3. Нажимает кнопку "Завершить"
4. Система фиксирует время окончания аренды
5. Система изменяет статус самоката на "доступен"

### 5. Ограничения и допущения

**5.1. Технические ограничения**
- Система рассчитана на 1000+ пользователей
- Время ответа API не более 2 секунд
- Поддержка мобильных браузеров

**5.2. Бизнес-ограничения**
- Один пользователь может иметь только одну активную аренду
- Аренда длится фиксированно 30 минут
- Отсутствие системы оплаты на первом этапе

### 6. Требования к интерфейсу

**6.1. Веб-интерфейс**
- Адаптивный дизайн
- Интуитивно понятная навигация
- Акценты на уникальность самокатов

**6.2. Основные экраны**
- Главная страница с каталогом самокатов
- Страница "Мои аренды" с историей
- Базовая навигация между разделами
