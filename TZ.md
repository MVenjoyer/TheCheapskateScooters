# Лабораторная работа 1
## Техническое задание на разработку системы "TheCheapskateScooters"

### 1. Введение

**1.1. Назначение системы**  
Веб-приложение для аренды тематических самокатов. Система позволяет арендовать самокаты с уникальным дизайном (WOT, пивные банки в колесах и другие креативные варианты).

**1.2. Цели создания**
- Автоматизация процесса аренды тематических самокатов
- Предоставление уникального пользовательского опыта
- Упрощение поиска и бронирования креативных самокатов

### 2. Функциональные требования

**2.1. Просмотр каталога самокатов**
- Отображение списка всех самокатов с их дизайном
- Фильтрация по статусу (доступен/занят)
- Фильтрация по адресу точки аренды
- Отображение информации о самокате (ID, модель, тип дизайна, статус, адрес)

**2.2. Управление арендой**
- **Начало аренды**: Пользователь может взять доступный самокат в аренду
- **Завершение аренды**: Пользователь может завершить активную аренду
- Автоматическое обновление статуса самоката

**2.3. История аренд**
- Просмотр истории всех аренд пользователя
- Отображение деталей аренды (самокат, время начала, время окончания, статус)

### 3. Пользовательские роли

**3.1. Клиент**
- Просмотр каталога самокатов
- Начало и завершение аренды
- Просмотр истории своих аренд

### 4. Бизнес-процессы

**4.1. Процесс "Аренда самоката"**
1. Пользователь просматривает каталог самокатов
2. Применяет фильтры по статусу и адресу
3. Выбирает доступный самокат
4. Нажимает кнопку "Арендовать"
5. Система создает запись об аренде
6. Система изменяет статус самоката на "занят"

**4.2. Процесс "Завершение аренды"**
1. Пользователь переходит в раздел "Мои аренды"
2. Находит активную аренду
3. Нажимает кнопку "Завершить"
4. Система фиксирует время окончания аренды
5. Система изменяет статус самоката на "доступен"

### 5. Технические требования

**5.1. Технологический стек**
- **Бэкенд**: Django + Django REST Framework
- **Фронтенд**: React/Vue.js (опционально, можно начать с Django Templates)
- **База данных**: PostgreSQL
- **API**: RESTful API с JSON форматом
- **Аутентификация**: JWT токены

**5.2. Архитектурные требования**
- Разделение на три слоя: API, бизнес-логика, доступ к данным
- Использование Django ORM для работы с БД
- Применение Django REST Framework сериализаторов

**5.3. Производительность**
- Время ответа API не более 500 мс для 95% запросов
- Поддержка 1000+ одновременных пользователей

**5.4. Безопасность**
- HTTPS для всех соединений
- Валидация входных данных на всех уровнях
- Защита от SQL-инъекций через Django ORM
- Хеширование паролей через Django Auth

### 6. Ограничения проекта

**6.1. Технические ограничения**
- Первая версия - только веб-интерфейс (без мобильных приложений)
- Отсутствие интеграции с платежными системами в MVP

**6.2. Бизнес-ограничения MVP**
- Один пользователь - одна активная аренда
- Фиксированная длительность аренды (30 минут)
- Лимит на количество самокатов в точке аренды
- Ручная модерация контента (дизайнов самокатов)

**Стек разработки:**
- Django 4.2+
- Django REST Framework
- PostgreSQL 14+
- Docker (для развертывания)

**План разработки (MVP):**
1. Настройка Django проекта
2. Создание моделей и миграций
3. Реализация REST API
4. Базовая админ-панель Django
5. Простые шаблоны для пользовательского интерфейса
6. Аутентификация (JWT)
7. Тестирование API
8. Деплой
